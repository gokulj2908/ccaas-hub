## Phase 6: True Omnichannel & The Closed Loop (Agent-to-Customer Chat)

**Architect:** Gokul J.
**Objective:** Break the "one-way glass" of standard integrations. Build a real-time, bidirectional conversational interface that allows a human executive inside Salesforce to chat seamlessly with a customer on WhatsApp, maintaining a strict omnichannel state.

## 🏗️ 1. The Agent "Send" Architecture (CRM to Edge)

### Concept
In an enterprise environment, agents should never leave their system of record (the CRM) to reply to a customer. We built a low-code UI component that sits directly on the Salesforce Case, allowing the agent to type a message that instantly routes to the customer's WhatsApp.

### Technical Implementation
* **Salesforce Screen Flow UI:** Designed a custom Screen Flow component embedded directly onto the Lightning Case Record Page using the `recordId` variable for dynamic context.
* **Declarative HTTP Callouts:** Configured a synchronous HTTP Callout within the Flow, mapping the agent's input and the Contact's phone number to an `Apex-Defined` JSON payload (`AgentPayloadVar`).
* **NestJS Middleware Endpoint:** Built a dedicated `POST /whatsapp/agent-reply` endpoint. 
* **Defensive Programming:** Implemented strict null-checks to catch bad CRM data (e.g., Cases missing a Contact lookup) *before* it crashes the Node.js server or the Twilio REST API.

### Architectural Challenges Solved
* **Headless UI Integration:** Proved the ability to decouple the frontend UI (Salesforce Flow) from the execution engine (NestJS), allowing the CRM to act purely as a presentation layer while the middleware handles telecommunications.
* **Bypassing the Cache:** Mastered Lightning App Builder assignment hierarchies (App Default vs. Org Default) to override aggressive browser caching and enforce new UI layouts.

## 🔄 2. The Interactive Loop (Edge to CRM)

### Concept
When the customer replies to the human agent, the middleware must recognize that the user is no longer talking to the AI. It must bypass the standard IVR logic and push the message directly to the agent's screen in real-time.

### Technical Implementation
* **State Machine Handoff:** Updated the Redis-backed state machine with a dedicated `WITH_AGENT` state. When the bot hands off the conversation, it permanently anchors the session to the specific Salesforce `caseId`.
* **Salesforce REST API (Chatter Integration):** Instead of forcing the agent to manually refresh the page to see replies, the NestJS hub intercepts the inbound Twilio webhook and executes a `POST` request to the Salesforce `FeedItem` object.
* **WebSocket Exploitation:** By writing directly to the Case Chatter Feed, we leverage Salesforce's native WebSockets. The UI auto-updates instantly on the agent's screen without requiring custom Lightning Web Component (LWC) code.
